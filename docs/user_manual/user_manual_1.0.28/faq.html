Frequently Asked Questions


Will DICOM+ over-write my existing files?
No - The program is designed to preserve existing files. Both anonymization and editing create new copies of the files. Before a new file is written, the program checks to make sure that the name it is using is unique.
Note that if the same series is anonymized multiple times, it will result in multiple sets of files. This is by design, as a common use case is to anonymize the same set of files multiple times with different user IDs to create test data.

The only exceptions are:

  • When editing a file, the user specifically indicates that the original file should be over-written.
  • When the user repeatedly saves during an edit session, the same destination file is used.


How does DICOM+ construct file names?
File names are constructed so as to be human readable, and are formatted with the following fields (if they are present in the source DICOM file) separated by underscore:

  • Patient ID
  • Modality
  • Series Number
  • Instance Number
If the resulting file already exists, an integer is appended to the name to make it unique.
